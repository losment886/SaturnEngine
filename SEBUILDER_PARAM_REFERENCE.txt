│ 例子: -o "C:\My Output"                                             │

🔤 参数列表

需要值的参数（都支持引号）:
  -project, -p              项目/解决方案路径
  -output, -o               编译输出目录
  -packageoutput, -po       打包输出目录
  -cooksourcedir, -csd      资源源目录
  -cookoutputdir, -cod      资源输出目录
  -prerendersource, -prs    预渲染源文件
  -prerenderoutput, -pro    预渲染输出文件
  -compressionsource, -cms  压缩源文件
  -compressionoutput, -cmo  压缩输出文件

不需要值的参数:
  -help, -h                 显示帮助
  -aot                      启用 AOT 编译
  -includepdb               包含调试符号
  -verbose, -v              详细输出


📊 参数值速查

平台 (-plt, -plt=):
  win / windows             Windows
  mac / macos               macOS
  linux                     Linux
  android                   Android
  ios                       iOS

架构 (-a, -a=):
  x86                       32位
  x64 / amd64               64位
  arm64 / aarch64           ARM64位

配置 (-c, -c=):
  debug                     调试版本
  release                   发布版本
  shipping                  最终版本

命令 (第一个参数):
  build                     编译项目
  package                   打包
  cook                      资源烘焙
  prerender                 预渲染
  compress                  压缩

✅ 最佳实践

✓ 对所有路径都使用双引号，即使目前没有空格
  dotnet SEBuilder.dll build -o="./output" -plt=win

✓ 在脚本中使用等号格式保持一致
  dotnet SEBuilder.dll build -plt=win -a=x64 -c=release -o="output"

✓ 在 PowerShell 脚本中使用变量
  dotnet SEBuilder.dll build -o "$env:BUILD_OUTPUT" -plt=win

✓ 对长命令使用续行符
  dotnet SEBuilder.dll build \
    -project="path/to/project.sln" \
    -o="output directory" \
    -plt=win -a=x64


❌ 常见错误

✗ 遗漏双引号（路径中有空格）
  dotnet SEBuilder.dll build -o=C:\My Output  # ❌ 错误
  应该: dotnet SEBuilder.dll build -o="C:\My Output"  # ✅ 正确

✗ 混用不同格式
  dotnet SEBuilder.dll build -project="path" -o ./out -plt win  # ❌ 混乱
  应该: dotnet SEBuilder.dll build -project="path" -o="./out" -plt=win  # ✅ 一致

✗ 嵌套引号
  dotnet SEBuilder.dll build -o="C:\Output "Extra" Dir"  # ❌ 错误
  应该: 使用其他方式分隔或转义

✗ 忘记完成引号
  dotnet SEBuilder.dll build -o="C:\Output  # ❌ 缺少结束引号
  应该: dotnet SEBuilder.dll build -o="C:\Output"  # ✅ 配对完整

📞 遇到问题?

问题: 参数值中的引号没有被移除
解决: 确保在最后一个字符前后加了引号
      正确: -o="path"    # 引号会被移除
      错误: -o="path     # 引号不匹配

问题: 路径中有 & 或 | 等特殊字符
解决: 用双引号括起整个路径
      dotnet SEBuilder.dll build -o="C:\Path & More\Output"

问题: 在 Bash 中无法正确传递参数
解决: 确保单引号和双引号配对正确
      dotnet SEBuilder.dll build -o='C:\My Path' -plt=linux
